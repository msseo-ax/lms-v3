# HOMES 사내 LMS - Product Requirements Document

## 1. 프로젝트 컨텍스트

- **목적**: 사내 정보 전달 및 열람 확인을 위한 Learning Management System
- **배경**: 기존에 노션·슬랙으로 자료를 공유하고 구글 드라이브에 보관하는 방식 → 열람 추적 불가, 대상 지정 불가, 정보 분산 문제
- **예상 사용자**: 50~100명 (전 사원)
- **주 사용 환경**: 데스크톱 + 모바일 (출퇴근 시 모바일 소비 고려)
- **콘텐츠 특성**: AI 생성 결과물이 많음 (AI 대화 내용, NotebookLM 영상 등 사내 지식 가공물)

---

## 2. 기술 스택 & 제약조건

| 항목 | 선택 |
|------|------|
| 프레임워크 | Next.js |
| DB | Supabase (PostgreSQL) |
| 스토리지 | AWS S3 |
| 인증 | Supabase Auth + Google OAuth |
| 호스팅 | Vercel |
| 네트워크 | 외부 접근 가능 |
| 도메인 | 보유 (미연결) |

---

## 3. 사용자 역할

### 관리자 (Admin)
- 콘텐츠 등록 및 삭제
- 콘텐츠 등록 시 대상을 구체적으로 설정 (전체/본부/팀/개인/복수 선택)
- 열람 현황 확인
- 미열람자 대상 슬랙 리마인드 발송

### 사용자 (User)
- 콘텐츠 열람 및 읽음 처리
- 콘텐츠 대상 확인 (누구를 위한 콘텐츠인지)
- 내가 대상인 콘텐츠 우선 노출
- 카테고리·대상 기반 필터링

---

## 4. 핵심 기능 (우선순위)

### P0 — 이번에 반드시 구현

- **콘텐츠 등록/삭제**: PDF, DOCX, MP4, 이미지, 텍스트(리치텍스트), 외부 링크 지원. 1콘텐츠에 다중 파일 첨부 가능.
- **대상 지정**: 전체 / 본부별 / 팀별 / 개인별 / 복수 선택
- **카테고리 태깅**: 사내 비전/문화, 업무 매뉴얼, 정책/규정, 시장 리서치/인사이트, 온보딩, 공지사항 (관리자가 카테고리 추가/수정 가능)
- **콘텐츠 목록 (홈 피드)**: 내 대상 콘텐츠 우선 노출, 카테고리·대상 뱃지 표시, 읽음/미읽음 상태 표시
- **콘텐츠 상세 열람**: 파일 뷰어(PDF, 이미지 인라인 표시), 영상 재생, 링크 임베드, 다중 첨부 파일 목록
- **AI 요약 자동 생성**: 콘텐츠 등록 시 "직접 작성 / AI 생성" 토글 제공
- **읽음 확인**: 콘텐츠 단위로 추적 (상세 페이지 진입 + 체류시간 기준). 개별 파일 클릭은 부가 로그로 기록.
- **관리자 열람 현황 대시보드**: 콘텐츠별 열람율, 미열람자 목록
- **인증**: Google OAuth (@homes.global 도메인만 허용, 그 외 차단 + 안내 메시지)

### P1 — 바로 다음

- **슬랙 리마인드 알림**: 미열람자 대상 슬랙 메시지 발송
- **PWA 설정 & 모바일 반응형**: 홈(피드), 콘텐츠 상세, 마이페이지 순서로 모바일 최적화
- **콘텐츠 검색 & 필터링**: 키워드 검색, 카테고리 필터, 대상 필터

### P2 — 나중에

- 서비스 내 알림 센터
- 열람 통계 리포트 (팀별·본부별 열람율)
- 댓글/질문 기능
- 관리자 패널에서 권한 관리 (현재는 DB 직접 지정)

## 5. 화면 명세

### 사용자 화면

#### 5-1. 홈 (피드)
- **포함 요소**: 콘텐츠 카드 리스트
- **카드 구성**:
  ```
  [카테고리 태그]  [대상 뱃지: AMS팀 · PM팀]
  콘텐츠 제목
  AI 또는 직접 작성 요약 (2줄)
  등록일 · 읽음/미읽음 상태
  ```
- **정렬**: 내 대상 콘텐츠 우선 → 최신순
- **필터**: 카테고리별, 읽음/미읽음
- **이동**: 카드 탭 → 콘텐츠 상세

#### 5-2. 콘텐츠 상세
- **포함 요소**: 제목, 카테고리, 대상 범위, 등록일, 등록자, 요약, 본문(리치텍스트), 첨부 파일 목록 (인라인 뷰어 + 다운로드), 외부 링크 임베드
- **행동**: 페이지 진입 시 읽음 처리 시작 (체류시간 추적), 파일 클릭 시 개별 로그 기록
- **이동**: 뒤로 → 홈

#### 5-3. 마이페이지
- **포함 요소**: 내 프로필 (이름, 소속 본부/팀), 읽음/미읽음 콘텐츠 현황 (숫자), 미읽음 콘텐츠 목록
- **행동**: 미읽음 콘텐츠 탭 → 해당 상세 페이지 이동

### 관리자 화면 (데스크톱 전용)

#### 5-4. 콘텐츠 등록/편집
- **포함 요소**:
  - 제목 입력
  - 카테고리 선택 (드롭다운)
  - 대상 지정 (전체 / 본부 선택 / 팀 선택 / 개인 선택 / 복수 선택)
  - 본문 입력 (리치텍스트 에디터)
  - 요약 입력 (직접 작성 / AI 자동 생성 토글)
  - 파일 업로드 (다중, 최대 300MB/파일)
  - 외부 링크 추가
- **행동**: 등록 → 콘텐츠 관리 목록으로 이동

#### 5-5. 콘텐츠 관리 목록
- **포함 요소**: 전체 콘텐츠 테이블 (제목, 카테고리, 대상, 등록일, 열람율)
- **행동**: 행 클릭 → 편집, 삭제 버튼, 열람 현황 상세 이동

#### 5-6. 열람 현황 대시보드
- **포함 요소**: 콘텐츠별 열람율 (진행 바), 미열람자 명단, 리마인드 발송 버튼
- **행동**: 리마인드 버튼 → 미열람자에게 슬랙 알림 발송

## 6. 데이터 모델

### User
| 필드 | 타입 | 설명 |
|------|------|------|
| id | uuid | PK |
| email | string | @homes.global 계정 |
| name | string | 이름 |
| role | enum | admin / user |
| division_id | uuid | FK → Division |
| team_id | uuid | FK → Team |
| created_at | timestamp | 가입일 |

### Division (본부)
| 필드 | 타입 | 설명 |
|------|------|------|
| id | uuid | PK |
| name | string | 본부명 |

### Team (팀)
| 필드 | 타입 | 설명 |
|------|------|------|
| id | uuid | PK |
| name | string | 팀명 |
| division_id | uuid | FK → Division |

### Category
| 필드 | 타입 | 설명 |
|------|------|------|
| id | uuid | PK |
| name | string | 카테고리명 |
| created_at | timestamp | 생성일 |

### Content
| 필드 | 타입 | 설명 |
|------|------|------|
| id | uuid | PK |
| title | string | 제목 |
| body | text | 본문 (리치텍스트) |
| summary | text | 요약 (직접 작성 or AI 생성) |
| summary_type | enum | manual / ai |
| category_id | uuid | FK → Category |
| created_by | uuid | FK → User (등록자) |
| created_at | timestamp | 등록일 |
| updated_at | timestamp | 수정일 |

### ContentFile
| 필드 | 타입 | 설명 |
|------|------|------|
| id | uuid | PK |
| content_id | uuid | FK → Content |
| file_url | string | S3 URL |
| file_type | enum | pdf / docx / mp4 / image / link |
| file_name | string | 원본 파일명 |
| file_size | integer | 파일 크기 (bytes) |
| created_at | timestamp | 업로드일 |

### ContentTarget
| 필드 | 타입 | 설명 |
|------|------|------|
| id | uuid | PK |
| content_id | uuid | FK → Content |
| target_type | enum | all / division / team / user |
| target_id | uuid | nullable. Division/Team/User의 ID. all인 경우 null |

### ReadLog
| 필드 | 타입 | 설명 |
|------|------|------|
| id | uuid | PK |
| content_id | uuid | FK → Content |
| user_id | uuid | FK → User |
| read_at | timestamp | 최초 열람 시각 |
| duration_seconds | integer | 체류시간 (초) |

### FileAccessLog
| 필드 | 타입 | 설명 |
|------|------|------|
| id | uuid | PK |
| content_file_id | uuid | FK → ContentFile |
| user_id | uuid | FK → User |
| accessed_at | timestamp | 파일 열람 시각 |

## 7. PWA & 모바일 요구사항

- **PWA 범위**: manifest.json + 홈화면 설치 지원 (앱 아이콘, 전체화면, 테마 컬러). Service Worker는 기본 셸 캐싱만.
- **오프라인 지원**: 없음. 온라인 전제.
- **모바일 반응형 우선순위**: 홈(피드) → 콘텐츠 상세 → 마이페이지 순서.
- **관리자 화면**: 데스크톱 전용 (모바일 반응형 불필요)
- **푸시 알림**: PWA 푸시 없음. 슬랙으로 대체.

---

## 8. 인증 & 보안

- **인증 방식**: Google OAuth (Supabase Auth)
- **허용 도메인**: @homes.global만 허용. 그 외 도메인 로그인 시도 시 차단 + "회사 계정으로 로그인해주세요" 안내.
- **권한 관리**: User 테이블의 role 필드를 Supabase DB에서 직접 지정 (admin / user)
- **세션**: Supabase Auth 기본 세션 관리

## 9. 비기능 요구사항

| 항목 | 기준 |
|------|------|
| 동시 접속 | 최대 100명. Vercel + Supabase Pro 플랜으로 충분 |
| 파일 업로드 제한 | 파일당 최대 300MB |
